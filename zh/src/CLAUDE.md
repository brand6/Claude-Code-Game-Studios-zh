# 源代码目录

在此目录中编写或修改游戏代码时，请遵循以下标准。

## 引擎版本警告

LLM 的训练数据早于当前固定的引擎版本。
**在使用任何引擎 API 之前，务必先查阅 `docs/engine-reference/`。**
不要猜测截止日期后的 API 签名——请先查文档。

## 编码标准

- 所有公开 API 必须添加文档注释
- 游戏数值必须**由数据驱动**（外部配置文件），绝不硬编码
- 优先使用依赖注入而非单例，以提高可测试性
- 每个新系统都需要在 `docs/architecture/` 中创建对应的 ADR
- 提交必须引用相关的 Story ID 或设计文档

## 文件路由

根据所编写文件的类型匹配对应的引擎专员 Agent。
参见 `CLAUDE.md` → 技术偏好配置 → 引擎专员 → 文件扩展名路由。

不确定时，使用 `CLAUDE.md` 中配置的主要引擎专员。

## 测试

测试文件位于 `tests/`——而非 `src/`。
如果测试框架尚未搭建，运行 `/test-setup` 进行脚手架初始化。
每个游戏系统都应有覆盖其公式和边界情况的单元测试。

## 验证驱动开发

添加游戏系统时先编写测试。
对于 UI 变更，通过截图进行验证。
将预期输出与实际输出对比，确认无误后再标记工作完成。
