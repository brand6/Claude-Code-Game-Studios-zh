# Agent 测试规格：art-director

## Agent 摘要
**拥有领域：** 视觉身份认同、美术圣经创作与执行、资产质量标准、UI/UX 视觉设计、视觉阶段关卡、概念艺术评估。
**不拥有：** UX 交互流程和信息架构（属于 ux-designer 的领域）、音频方向（audio-director）、代码实现。
**模型层级：** Sonnet（注意：尽管有"director"头衔，art-director 按 coordination-rules.md 分配 Sonnet — 它处理单个系统分析，而非 Opus 级别的多文档阶段关卡综合）。
**处理的关卡 ID：** AD-CONCEPT-VISUAL、AD-ART-BIBLE、AD-PHASE-GATE。

---

## 静态断言（结构性）

通过读取 agent 的 `.claude/agents/art-director.md` frontmatter 验证：

- [ ] `description:` 字段存在且针对特定领域（引用视觉身份、美术圣经、资产标准 — 而非通用描述）
- [ ] `allowed-tools:` 列表以读取为主；支持时包含图像评审能力；除非有资产管线检查的合理需求否则不包含 Bash
- [ ] 模型层级为 `claude-sonnet-4-6`（非 Opus — coordination-rules.md 将 art-director 分配为 Sonnet）
- [ ] Agent 定义不声明对 UX 交互流程或音频方向的权限

## 测试用例

### 用例 1：领域内请求 — 适当的输出格式
**场景：** 美术圣经的调色板章节提交审查。该章节定义了与游戏支柱"腐朽之美"挂钩的高对比点缀色去饱和大地色系主调色板。调色板内部一致，并引用了支柱词汇。请求标记为 AD-ART-BIBLE。
**预期：** 返回 `AD-ART-BIBLE: APPROVE`，附理由确认调色板的内部一致性及其与所述支柱的对齐。
**断言：**
- [ ] 裁决恰好为 APPROVE / CONCERNS / REJECT 之一
- [ ] 裁决令牌格式为 `AD-ART-BIBLE: APPROVE`
- [ ] 理由引用具体的调色板特征和支柱对齐 — 而非通用美术建议
- [ ] 输出保持在视觉领域内 — 不评论 UX 交互模式或音频氛围

### 用例 2：领域外请求 — 重定向或升级
**场景：** 音效设计师请求 art-director 说明当玩家进入战斗区域时环境音频应如何分层和闪避。
**预期：** Agent 拒绝定义音频行为，并重定向至 audio-director。
**断言：**
- [ ] 不对音频分层或闪避行为做出任何约束性决策
- [ ] 明确指明 `audio-director` 为正确的处理方
- [ ] 可以指出音频是否有视觉氛围影响（例如"音频应与区域的视觉张力相匹配"），但将所有音频规格交由 audio-director

### 用例 3：关卡裁决 — 正确的裁决词汇
**场景：** 主角的概念艺术提交审查。作品使用了鲜艳饱和的配色方案（主色：#FF4500、#00BFFF），与美术圣经中既定的"去饱和大地色系"调色板规范直接冲突。请求标记为 AD-CONCEPT-VISUAL。
**预期：** 返回 `AD-CONCEPT-VISUAL: CONCERNS`，具体引用调色板差异，对比美术圣经中声明的调色板值与提交概念作品的调色板。
**断言：**
- [ ] 裁决恰好为 APPROVE / CONCERNS / REJECT 之一 — 而非自由文本
- [ ] 裁决令牌格式为 `AD-CONCEPT-VISUAL: CONCERNS`
- [ ] 理由具体指出调色板冲突 — 而非"与风格不匹配"的泛泛评论
- [ ] 将美术圣经作为正确调色板的权威来源引用

### 用例 4：冲突升级 — 正确的上级
**场景：** ux-designer 提议在 HUD 中使用高对比度、色彩鲜艳的图标以提高可读性。art-director 认为这违反了美术圣经中柔和的视觉语言，会破坏视觉身份认同。
**预期：** art-director 陈述视觉身份顾虑并引用美术圣经，承认 ux-designer 的可读性目标是合理的，并升级至 creative-director 来仲裁视觉一致性与可用性之间的权衡。
**断言：**
- [ ] 升级至 `creative-director`（创意领域冲突的共同上级）
- [ ] 不单方面推翻 ux-designer 的可读性建议
- [ ] 将冲突清晰定义为两个合理目标之间的权衡
- [ ] 引用被违反的具体美术圣经规则

### 用例 5：上下文传递 — 使用所提供的上下文
**场景：** Agent 收到一个关卡上下文块，包含现有美术圣经的具体调色板值（主色：#8B7355、#6B6B47；点缀色：#C8A96E）和风格规则（"无纯白，无纯黑；所有阴影带暖色调"）。提交了一个新资产供审查。
**预期：** 评估引用所提供美术圣经中的具体色值和风格规则，而非通用色彩理论建议。任何顾虑都与所提供规则的具体违反相关联。
**断言：**
- [ ] 引用所提供美术圣经上下文中的具体调色板值
- [ ] 应用所提供文档中的具体风格规则（无纯白/黑、暖色阴影）
- [ ] 不生成与所提供美术圣经脱节的通用美术指导反馈
- [ ] 裁决理由可追溯至所提供上下文中的具体行或规则

## 协议合规性

- [ ] 仅使用 APPROVE / CONCERNS / REJECT 词汇返回裁决
- [ ] 严守声明的视觉领域
- [ ] 将 UX 与视觉冲突升级至 creative-director
- [ ] 在输出中使用关卡 ID（例如 `AD-ART-BIBLE: APPROVE`），而非内联散文裁决
- [ ] 不做约束性 UX 交互、音频或代码实现决策
