# Unreal Engine 5.7 — PCG（程序化内容生成）

**最后验证：** 2026-02-13
**状态：** 生产就绪（UE 5.7 起）
**插件：** `PCG`（内置，在 Plugins 中启用）

---

## 概述

**程序化内容生成（PCG）** 是 Unreal 的节点式框架，用于大规模程序化生成内容。专为在广阔开放世界中填充植被、岩石、道具、建筑和其他环境细节而设计。

**以下场景使用 PCG：**
- 程序化植被放置（树木、草地、岩石）
- 基于生物群系的环境生成
- 道路/路径生成
- 建筑/结构放置
- 世界细节填充（道具、杂物）

**以下场景不要使用 PCG：**
- 游戏逻辑（使用 Blueprint/C++）
- 需要手动精确摆放的单个物件（使用编辑器工具）

**⚠️ 说明：** PCG 在 UE 5.0-5.6 中为实验性功能，UE 5.7 起正式进入生产就绪状态。

---

## 核心概念

### 1. **PCG 图（PCG Graph）**
- 节点式图（类似材质编辑器）
- 定义生成规则

### 2. **PCG 组件（PCG Component）**
- 放置在关卡中，执行 PCG 图
- 在指定体积范围内生成内容

### 3. **PCG 数据（PCG Data）**
- 点数据（位置、旋转、缩放）
- 样条数据（路径、道路、河流）
- 体积数据（密度、生物群系蒙版）

### 4. **节点（Nodes）**
- **采样器（Samplers）**：生成点（网格、泊松、表面）
- **过滤器（Filters）**：按规则删除点（密度、标签、边界）
- **修改器（Modifiers）**：变换点（偏移、旋转、缩放）
- **生成器（Spawners）**：在点处实例化网格体/Actor

---

## 配置

### 1. 启用插件

`Edit > Plugins > PCG > Enabled > Restart`

### 2. 创建 PCG 体积

1. Place Actors > Volumes > PCG Volume
2. 缩放体积到所需生成区域

### 3. 创建 PCG 图

1. Content Browser > PCG > PCG Graph
2. 打开 PCG 图编辑器

---

## 基础工作流

### 示例：森林生成

#### 1. 创建 PCG 图

**节点设置：**
```
输入（体积）
  ↓
表面采样器（采样体积表面，每平方米点数：0.5）
  ↓
密度过滤器（使用贴图蒙版或噪声）
  ↓
静态网格体生成器（树木网格体）
  ↓
输出
```

#### 2. 将图指定给体积

1. 选择 PCG 体积
2. Details Panel > PCG Component > Graph = 你的 PCG 图
3. 点击 "Generate" 按钮

## 主要节点类型

### 采样器（点生成）

#### 网格采样器
- 规则网格排列的点
- 配置：
  - **网格大小（Grid Size）**：点与点之间的距离
  - **偏移（Offset）**：每个点的随机偏移

#### 泊松盘采样器
- 具有最小间距的随机点
- 配置：
  - **每平方米点数（Points Per m²）**：密度
  - **最小距离（Min Distance）**：点与点之间的间距

#### 表面采样器
- 网格体表面或地形上的点
- 配置：
  - **每平方米点数（Points Per m²）**：密度
  - **仅表面（Surface Only）**：仅在表面，不含体积内部

---

### 过滤器（点移除）

#### 密度过滤器
- 根据密度值移除点
- 输入：贴图或噪声
- 用途：生物群系蒙版、空地、路径

#### 标签过滤器
- 按标签过滤点
- 用途：条件性生成

#### 边界过滤器
- 仅保留边界范围内的点
- 用途：将生成限制在特定区域

---

### 修改器（点变换）

#### 旋转
- 随机化点的旋转
- 配置：
  - **最小/最大旋转（Min/Max Rotation）**：每个轴的旋转范围

#### 缩放
- 随机化点的缩放
- 配置：
  - **最小/最大缩放（Min/Max Scale）**：缩放范围

#### 投影到地面
- 将点吸附到地形表面

---

### 生成器（网格体/Actor 实例化）

#### 静态网格体生成器
- 在点处生成静态网格体
- 配置：
  - **网格体列表（Mesh List）**：网格体数组（随机选取）
  - **裁剪距离（Culling Distance）**：LOD/裁剪设置

#### Actor 生成器
- 在点处生成 Blueprint Actor
- 用途：游戏逻辑 Actor、可交互对象

---

## 数据源

### 地形（Landscape）
- 以地形作为采样输入
- 自动投影到地形高度

### 样条线（Splines）
- 沿样条线生成内容（道路、河流、路径）
- 示例：沿路径种树

### 贴图（Textures）
- 以贴图作为密度蒙版
- 绘制生物群系、空地、区域

## 生物群系示例（混合森林）

### 图设置

```
输入（地形）
  ↓
表面采样器（密度：1.0）
  ↓
┌─────────────────┬─────────────────┐
│ 树木生物群系      │ 岩石生物群系      │
│ （密度 > 0.5）   │ （密度 < 0.5）   │
├─────────────────┼─────────────────┤
│ 树木生成器        │ 岩石生成器        │
└─────────────────┴─────────────────┘
  ↓
合并
  ↓
输出
```

## 基于样条线的生成（带树木的道路）

### 1. 创建 PCG 图

```
样条线输入
  ↓
样条线采样器（沿样条线采样）
  ↓
偏移（从样条线路径侧移）
  ↓
树木生成器
  ↓
输出
```

### 2. 为 PCG 体积添加样条线组件

1. PCG Volume > Add Component > Spline
2. 绘制样条线路径
3. PCG 图读取样条线数据

## 运行时生成

### 通过 C++ 触发生成

```cpp
#include "PCGComponent.h"

UPCGComponent* PCGComp = /* 获取 PCG 组件 */;
PCGComp->Generate(); // 执行 PCG 图
```

### 流式生成（大型世界）

- PCG 与 World Partition 自动流式加载
- 仅在已加载的分区格中生成内容

---

## 性能

### 优化建议

- 为生成的网格体设置**裁剪距离**（LOD）
- 限制**密度**（点越少性能越好）
- 对重复网格体使用**层级实例化静态网格体（HISM）**
- 为大型世界启用**流式加载**

### 调试性能

```cpp
// 控制台命令：
// pcg.graph.debug 1 - 显示 PCG 调试信息
// stat pcg - 显示 PCG 性能统计
```

## 常用模式

### 带空地的森林

```
表面采样器
  ↓
密度过滤器（带空地的噪声贴图）
  ↓
树木生成器（松树、橡树、桦树）
```

---

### 陡坡上的岩石

```
地形输入
  ↓
表面采样器
  ↓
坡度过滤器（角度 > 30°）
  ↓
岩石生成器
```

---

### 道路旁的道具

```
样条线输入（道路样条线）
  ↓
样条线采样器
  ↓
偏移（路边）
  ↓
路灯生成器
```

---

## 局限性

- **不适用于游戏逻辑**：游戏规则请使用 Blueprint/C++
- **大型图可能较慢**：使用过滤器和降低密度来优化
- **运行时生成有开销**：尽可能预先生成
