# 设计文档规则

- 每份设计文档必须包含以下 8 个章节：概述、玩家幻想、详细规则、公式、边界情况、依赖关系、调节旋钮、验收标准
- 公式必须包含变量定义、预期取值范围和示例计算
- 边界情况必须明确说明发生了什么，而非仅写"妥善处理"
- 依赖关系必须双向记录——若系统 A 依赖 B，则 B 的文档中也必须提及 A
- 调节旋钮必须指定安全范围及其影响的游戏体验维度
- 验收标准必须可测试——QA 测试人员必须能够验证通过/失败
- 不得含糊其辞："系统感觉应该不错"不是有效的规格说明
- 平衡数值必须关联来源公式或其合理性依据
- 设计文档必须增量撰写：先创建骨架，然后逐节填写内容，每节须经用户确认后方可继续。每个已确认的章节须立即写入文件，以持久化决策并管理上下文
