# 发现与决策

## 需求

- 将 CCGS 从"用户主导协作"模式迁移为"用户主导决策 + AI 自主执行"模式
- 核心理念：前期问清楚需求，后期自主交付
- 输出到 `zh-auto/` 目录，保持与 `zh/` 相同子目录结构

## 研究发现

### 原版文档分析（zh/docs/COLLABORATIVE-DESIGN-PRINCIPLE.md）

- 约 700 行，核心理念是"每步都审批"
- 工作流为 11 步：提问→选项→决策→草稿→迭代→最终审批→写入
- 关键规则："未经明确批准，绝不写入文件"
- 包含三类示例：设计任务、编码任务、头脑风暴
- 有价值的保留内容：AskUserQuestion 指南、提问方式指南、增量章节写入
- 需大幅改写的内容：核心理念、工作流、文件写入协议、Agent 个性

### 前次讨论确认的优化建议

1. 显式划分决策区/执行区
2. 需求冻结确认（进入执行前的边界协商）
3. 上报协议（三级：静默/通知/阻塞）
4. AI 内部质量门禁（替代原版逐步审批）
5. Review 循环加终止条件
6. 结构化验收交付物
7. 快速通道（不同任务类型的简化路径）

## 技术决策

| 决策 | 理由 |
|------|------|
| 重写而非微调 | 核心理念根本性变化 |
| 保留 AskUserQuestion | 决策阶段仍需结构化提问 |
| 保留增量章节写入 | 决策阶段的文档协作仍需要 |
| 标题模式补"选项"步骤 | 流程描述的准确性 |
