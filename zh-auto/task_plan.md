# 任务计划：CCGS → Auto 模式迁移

## 目标
将 Claude Code Game Studios（zh 版）从"每步审批的协作模式"迁移为"用户主导决策、AI 自主执行"的自动化开发框架。

## 核心理念
- 用户主导创意方向和关键决策
- AI 主导开发实现和质量保证
- 前期充分沟通需求，后期自主交付成果

## 关键决策
| 决策 | 理由 |
|------|------|
| 重写而非微调 | 核心理念根本性变化，逐句修改会导致新旧理念混杂 |
| 保留 AskUserQuestion 指南 | 决策阶段仍需结构化提问 |
| 保留增量章节写入 | 决策阶段的文档协作仍需要此机制 |
| 去掉"不是自主执行者" | 新框架下 AI 在执行阶段就是自主执行者 |
| 用户画像放两处 | CLAUDE.md 简版（一句话）+ 协作原则完整版（画像+翻译规则+示例）|
| 冻结后变更走 /change-request | Phase 4 新增技能，复用 propagate-design-change + scope-check |
| 快速通道与 /change-request 不重复 | 快速通道=任务入口路由，/change-request=执行中变更控制 |
| Review 必须生成报告文件 | 报告落盘避免第2轮重复审查，同时作为验收证据 |
